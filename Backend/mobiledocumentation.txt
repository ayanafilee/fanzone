================================================================================
                        MOBILE APP INTEGRATION GUIDE
================================================================================

RECOMMENDED FLOW FOR PUBLIC APP (NO LOGIN):

1. APP LAUNCH
   - No authentication needed
   - Load default language (device language or English)
   - Navigate to home screen

2. ONBOARDING (OPTIONAL)
   Step 1: GET /api/languages ‚Üí Show language selector
   Step 2: GET /api/clubs ‚Üí Show club selector (optional favorite)
   Step 3: Save preferences locally on device
   Step 4: Navigate to home screen

3. HOME SCREEN (MAIN FEED)
   - GET /api/feed/all
   - Display feed items in list/card view
   - Pull-to-refresh to reload feed
   - Tap news item ‚Üí GET /api/news/:id ‚Üí Show detail view
   - Tap highlight ‚Üí GET /api/highlights/:id ‚Üí Play video

4. CLUBS TAB
   - GET /api/clubs
   - Display clubs grouped by league
   - Tap club ‚Üí Filter content by club_id
   - GET /api/content?club_id=xxx
   - GET /api/highlights?club_id=xxx

5. WATCH TAB
   - GET /api/watch-platforms
   - Display as grid of platform cards
   - Tap platform ‚Üí Open URL in browser/webview

6. SETTINGS
   - GET /api/languages ‚Üí Show language selector
   - Save language preference locally
   - Reload content with new language

7. MULTILINGUAL CONTENT
   - News articles have title/body in en, am, om
   - Display based on user's language preference (stored locally)
   - Example: user selected "am" ‚Üí show title.am, body.am
   - Fallback to English if translation missing

8. LOCAL STORAGE
   - Store user preferences locally (language, favorite club)
   - No server-side user accounts needed
   - Use SharedPreferences (Android) or UserDefaults (iOS)

================================================================================
                          ADDITIONAL NOTES
================================================================================

1. All timestamps are in ISO 8601 format (UTC)
   Example: "2024-01-15T10:30:00Z"

2. MongoDB ObjectIDs are 24-character hexadecimal strings
   Example: "507f1f77bcf86cd799439011"

3. Image URLs and video URLs are full URLs
   Load them directly in image views or video players

4. Empty/null values:
   - Empty strings: ""
   - Missing fields may not be present in response
   - Check for null/undefined before accessing nested properties

5. Rate limiting: Not currently implemented
   Consider implementing on client side to avoid excessive requests

6. Pagination: Not currently implemented
   All endpoints return complete results
   Consider implementing if performance issues arise

7. Caching recommendations:
   - Cache club list (rarely changes)
   - Cache language list (static)
   - Cache user preferences locally (language, favorite club)
   - Cache feed for short duration (5-10 minutes)
   - Implement pull-to-refresh for manual updates

8. Network error handling:
   - Implement retry logic for failed requests
   - Show user-friendly error messages
   - Provide offline mode with cached content if possible

9. Security best practices:
   - Always use HTTPS in production
   - Validate all data before displaying
   - Sanitize URLs before opening in webview
   - Implement certificate pinning if possible

10. Performance optimization:
    - Load images lazily in feed
    - Implement infinite scroll/pagination for long feeds
    - Compress images before displaying
    - Cache API responses appropriately
